CLI-Guide.JS

A javascript library for creating interactive command line tutorials that run in your web browser

Demo

How do I use it?


Initial configurarion of the plugin, is necessary to use bootstrap.min.css is required because the plugin uses a Bootstrap CSS grid 


                  
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="utf-8" />
  <title>Cli Guide JS</title>
  <link rel="stylesheet" type="text/css" href="./src/bootstrap.min.css" />
  <link rel="stylesheet" type="text/css" href="./src/cli-guide.css" />
</head>
<body>
  <div id="stepsdiv"></div>
  <script type="text/javascript" src="./libs/jquery-1.11.3.min.js"></script>
  <script type="text/javascript" src="./src/cli-guide.js"></script>
  <script type="text/javascript" >
                  
                  
    
    $('#stepsdiv').cliguide(
    {
      nameOfTheProject: 'Apache Aurora',
      stepsFile:        'templates/apache_aurora.json',
      initStep: 1
    }).cli(
    {
      commandStepsFile: 'templates/apache_aurora_commands.json',
      preloadfile: 'templates/apache_aurora_files.json'
    }, function(){});
    
                  
                  
  </script>
</body>
</html>

How does it work?


This plugin uses json template for load Steps and Commands you can run in the terminal

A json template of steps apache_aurora.json 


                  
[
  {
  "step": "1",
    "content": {
      "title": "Setup: Install Aurora",
      "content": [
          " You use the Aurora client and web UI to interact with Aurora jobs. ",
          " To install it locally, see vagrant.md ",
          " The remainder of this Tutorial assumes you are running ",
          " Aurora using Vagrant. Unless otherwise stated, ",
          " all commands are to be run from the root ",
          " of the aurora repository clone. "
          ],
      "tips": "",
      "commands": [
      ]
    }
  }
]

A json template of commands apache_aurora_commands.json 


                  
[
  {
    "command":"aurora",
    "result": [
      "usage: aurora [-h] [--version] {task,quota,update,cron,job,config,sla} ... <br/>",
      "<br/>",
      "optional arguments: <br/>",
      "<div class='codeofcommand'>-h, --help</div> show this help message and exit <br/>",
      "<div class='codeofcommand'>--version</div> show program's version number and exit <br/>",
      "commands: <br/>",
      "{task,quota,update,cron,job,config,sla} <br/>",
      "<div class='codeofcommand'>task</div> Work with a task running in an Apache Aurora cluster <br/>",
      "<div class='codeofcommand'>quota</div> Work with quota settings for an Apache Aurora cluster <br/>",
      "<div class='codeofcommand'>update</div> Interact with the aurora update service. <br/>",
      "<div class='codeofcommand'>cron</div> Work with entries in the aurora cron scheduler <br/>",
      "<div class='codeofcommand'>job</div> Work with an aurora job <br/>",
      "<div class='codeofcommand'>config</div> Work with an aurora configuration file <br/>",
      "<div class='codeofcommand'>sla</div> Work with SLA data in Aurora cluster."
    ]
  }
]



You can style the result 

                  
<div class='codeofcommand'>task</div>

/* style for templates */
.codeofcommand {
  padding-left: 20px;
  width: 100px;
  height: auto;
  float: left;
}

For example if multiple commands return the same result. You can do this 


                  
"command":["vagrant", "vagrant list-commands"]
